# M10.7 v6b. 원소반응 KO/교체 연출 안정화

## 수정 1. 자동교체 타이밍 지연

문제:

- 남은 대기 펫이 1마리일 때 active 펫이 원소반응으로 쓰러지면 즉시 다음 펫으로 교체되었다.
- 그 결과 원소반응 플래시/데미지 숫자가 다음 펫에게 들어간 것처럼 보였다.

수정:

- 대기 펫이 1마리뿐이어도 KO 직후에는 즉시 participant를 바꾸지 않는다.
- 먼저 쓰러진 펫을 `switching` 상태에서 잠깐 보여준다.
- `switching` pause가 끝날 때 자동교체를 실제 반영한다.

## 수정 2. 원소반응 뱃지 중복 방지

문제:

- 원소반응으로 KO가 발생하면 같은 로그를 유지한 채 상태/turnStartTime이 바뀌면서 반응 뱃지가 다시 뜰 수 있었다.

수정:

- 반응 뱃지 중복 체크 기준을 `turnStartTime + log`가 아니라 `log` 자체로 변경했다.

## 수정 3. 교체 시 HP 차이 숫자 제거

문제:

- 펫이 교체될 때 이전 펫 HP와 다음 펫 HP 차이가 데미지/회복 숫자로 표시될 수 있었다.

수정:

- HP 변화 숫자는 같은 pet id일 때만 표시한다.
- active pet id가 바뀐 경우에는 피해/회복 숫자를 띄우지 않는다.
