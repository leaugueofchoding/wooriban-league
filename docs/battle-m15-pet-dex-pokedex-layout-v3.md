# M15. 도감형 펫 도감 레이아웃 v3b

## 목표

도감 화면을 작은 그리드 중심으로 정리하고, 선택한 펫 정보는 오른쪽 카드에서 압축해서 보여준다.

## 변경 내용

- 도감 번호를 No.001부터 시작하도록 변경
- 상태 색상 규칙 변경
  - 미발견: 실루엣/회색 칸
  - 우리 반 발견 + 내가 미보유: 초록색 칸
  - 내가 보유: 파란색 칸
  - 선택 중: 빨간 테두리
- 스킬 미리보기 기능 복구
- 왼쪽 그리드와 오른쪽 상세 카드 높이 맞춤
- 상세 카드는 태블릿 한 화면에 들어오도록 압축
- 기본 카드에서 전투 역할/상성/상성노트/발견한 친구 제거
- 하단 메뉴 동작 추가
  - 검색: 검색 입력창 열고 닫기
  - 상성노트: 원소반응/상성 노트 열고 닫기
  - 자세히: 전투 역할, 상성, 발견한 친구 표시
  - 닫는다: 오른쪽 상세 카드 숨김
- 상세 카드에 내용이 많아지면 카드 내부 스크롤로 처리


## v3d 조정

- 그리드 펫 아이콘 크기를 v2 수준으로 복구
- 오른쪽 상세 카드 가로폭을 약 60% 수준으로 축소
- 기본 상세 카드에서 진화조건 섹션 삭제
- 스킬 미리보기에서 시전자 펫 이미지는 정면(idle)이 아니라 뒷모습(battle)을 사용
- 뒷모습 이미지는 petImageMap의 `appearanceId_battle` 키를 우선 사용하고, 없으면 기존 정면 이미지를 fallback으로 사용


## v3e 조정

- 왼쪽 도감 그리드와 오른쪽 카드 비율을 v1/v2 계열로 복구
- v3d에서 오른쪽 카드 폭을 줄이면서 왼쪽 그리드가 지나치게 길어진 문제를 완화
- 발견한 사람은 자세히 버튼을 누르지 않아도 기본 카드에 표시
- 자세히/상성노트를 열어도 발견한 사람은 카드 맨 아래에 유지


## v3f 조정

- 도감 전체 max-width를 줄여 데스크톱에서 화면이 너무 넓게 벌어지지 않도록 조정
- 좌우 레이아웃을 비율 기반이 아니라 고정폭에 가까운 구조로 변경
  - 왼쪽 그리드: 약 500~560px
  - 오른쪽 카드: 약 300~360px
- 태블릿 가로 환경에서 세로로 쌓이는 것을 줄이기 위해 breakpoint를 860px로 낮춤
- 861~1080px 구간에서는 전용 compact 2열 규칙 적용
- 오른쪽 카드 내부 이미지 영역을 줄여 카드 폭 축소 후에도 세로 낭비를 줄임


## v3g 조정

- 스킬 미리보기는 더 이상 화면 오른쪽 아래 여백에 fixed로 뜨지 않는다.
- PetDexPage에서는 SkillPreview를 오른쪽 상세 카드 내부에 렌더링한다.
- SkillPreview에 `displayMode="cardOverlay"` 옵션을 추가했다.
- 카드 오버레이 모드에서는 오른쪽 상세 카드 위에 absolute overlay로 표시된다.
- 태블릿 환경에서도 스킬 미리보기가 상세 카드 영역을 중심으로 뜨도록 조정했다.


## v3h 조정

- 도감 스킬 미리보기 오버레이는 X 버튼뿐 아니라 빈 배경 클릭으로도 닫을 수 있다.
- 단, 스킬명/이미지/설명/버튼 등 내부 요소를 클릭할 때는 닫히지 않는다.
- 카드 오버레이 모드에서만 적용되며, 기존 fixed 모드 동작은 유지한다.


## v3i 조정

- 발견한 사람이 많은 경우 기본 화면에는 일부 이름만 표시한다.
- 숨겨진 인원은 `외 N명`으로 표시하되, N이 0이면 표시하지 않는다.
- 발견한 사람 영역에 마우스를 올리거나 키보드 포커스를 주면 전체 명단 오버레이가 표시된다.
- 학생들이 자기 이름을 확인할 수 있도록 전체 발견자 명단을 칩 형태로 노출한다.
