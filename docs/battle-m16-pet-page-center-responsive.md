# M16. 펫 페이지/펫센터 태블릿·모바일 압축 레이아웃

## 목표

펫 페이지와 펫센터가 태블릿 가로 화면, 작은 노트북 화면에서도 지나치게 길어지지 않도록 레이아웃을 압축한다.

## 적용 원칙

- 전투/펫 로직 변경 없음
- 데이터 구조 변경 없음
- styled-components의 크기, 간격, breakpoint만 조정
- 태블릿 가로 환경에서는 가능한 한 2열 유지
- 모바일에서는 1열로 자연스럽게 전환

## 펫 페이지 변경

- 전체 wrapper 여백 축소
- 메인 레이아웃을 1fr + 280px로 압축
- 900px 이하에서만 1열로 전환
- 펫 대시보드 padding/gap 축소
- 펫 이미지 크기 축소
- 정보 카드 padding/글자 크기 축소
- 스탯바 높이 축소
- 버튼 간격/높이 축소
- 보유 펫 목록 카드/이미지/간격 축소
- 보유 펫 목록 max-height 축소

## 펫센터 변경

- 전체 wrapper와 header 여백 축소
- 탭 버튼 높이/간격 축소
- content box padding 축소
- 상점 아이템 카드 grid 최소폭 축소
- 아이템 카드 padding/이미지/설명/버튼 크기 축소
- 치료소 펫 카드 grid 최소폭 축소
- 치료소 펫 카드 padding/이미지/상태/버튼 크기 축소
- 스킬 카드 padding 축소

## 확인 항목

- /pet 화면이 태블릿 가로 폭에서 너무 길지 않은지
- /pet에서 펫 이미지와 정보 카드가 과도하게 크지 않은지
- /pet 보유 펫 목록이 너무 길게 내려가지 않는지
- /pet-center 상점 카드가 한 화면에 더 많이 보이는지
- /pet-center 치료소 카드가 한 화면에 더 많이 보이는지
- 모바일에서는 1열로 깨지지 않는지
- npm run build 통과


## v2 조정

펫센터 상점과 치료소가 세로로 길어지는 문제를 줄이기 위해 카드 목록을 한 줄 가로 스크롤 구조로 변경했다.

### 변경 내용

- 상점 아이템 목록을 grid에서 horizontal scroll row로 변경
- 치료소 펫 목록을 grid에서 horizontal scroll row로 변경
- 태블릿/모바일 터치 스크롤을 위해 `-webkit-overflow-scrolling: touch` 적용
- 카드 단위로 멈추는 느낌을 위해 `scroll-snap-type: x mandatory` 적용
- hover 시 카드가 너무 크게 튀어 오르지 않도록 이동값 완화

### 판단

페이지 넘기기보다 가로 스크롤을 우선 적용한다.

- 구현이 단순함
- 상태 관리가 추가되지 않음
- 터치 태블릿에서 자연스러움
- 카드 수가 많아져도 한 화면 높이를 크게 늘리지 않음
