# M14. 원소반응 트리거 속성 상성 보정

## 목표

원소반응의 조합식과 CC는 유지하면서, 마지막에 반응을 터뜨린 스킬 속성에 따라 원소반응 추가 피해만 보정한다.

## 핵심 원칙

- 일반 스킬 피해에는 상성을 적용하지 않는다.
- 원소반응 추가 피해에만 상성을 적용한다.
- 원소반응 이름, CC, 상태효과는 바꾸지 않는다.
- 상성은 "마지막 공격 스킬 속성 vs 방어자 펫 속성"으로만 판단한다.
- 흔적을 먼저 묻힌 속성의 순서로 강약을 바꾸지 않는다.

## 상성 순환

물 > 불 > 풀 > 번개 > 바람 > 얼음 > 물

| 공격 스킬 속성 | 유리 | 불리 |
| --- | --- | --- |
| 물 | 불 | 얼음 |
| 불 | 풀 | 물 |
| 풀 | 번개 | 불 |
| 번개 | 바람 | 풀 |
| 바람 | 얼음 | 번개 |
| 얼음 | 물 | 바람 |

## 배율

| 판정 | 원소반응 추가 피해 |
| --- | ---: |
| 유리 | x1.20 |
| 보통 | x1.00 |
| 불리 | x0.80 |

## 예시

- 물 흔적 + 번개 스킬 = 감전
  - 감전 효과는 그대로
  - 번개 스킬로 터뜨렸으므로 번개 vs 방어자 속성으로 반응 추가 피해 보정

- 번개 흔적 + 물 스킬 = 감전
  - 감전 효과는 그대로
  - 물 스킬로 터뜨렸으므로 물 vs 방어자 속성으로 반응 추가 피해 보정

## 제외한 것

- CC 약화/강화 없음
- 기본 스킬 상성 없음
- 같은 반응의 효과 순서별 변경 없음


## v2 보정

기존 전투 문서의 pet 객체에는 element가 저장되어 있지 않고 species 또는 appearanceId만 있는 경우가 있다.

따라서 BattlePage에서 PET_DATA를 통해 방어자 펫 속성을 보강한 뒤, resolveElementReaction에 defenderElement를 명시 전달한다.

이 보정으로 기존 저장 데이터에서도 다음 로그가 정상 출력된다.

- 상성 유리!
- 상성 불리!


## v3 표시 보정

전투 raw log에는 상성 유리/불리 문구가 들어가도, 화면 표시용 compact log가 원소반응 줄만 남기면서 해당 문구를 숨길 수 있었다.

따라서 getCompactBattleReactionLines에서 다음 줄을 별도로 표시한다.

- 상성 유리! 반응 피해 증가!
- 상성 불리! 반응 피해 감소!
