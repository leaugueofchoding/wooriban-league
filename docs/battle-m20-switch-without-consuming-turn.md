# M20. 펫 교체 턴 비소모

## 목표

공격권을 얻은 플레이어가 펫을 교체해도 공격 기회를 잃지 않게 한다.

## 변경 전

- 공격자가 펫 교체 선택
- 교체 처리
- 공격 기회 소모
- 다음 퀴즈로 진행

## 변경 후

- 공격자가 펫 교체 선택
- 교체 처리
- 같은 action 턴 유지
- 공격자가 다시 스킬/기본공격 선택 가능
- 단, 같은 공격 턴에는 수동 교체 1회만 가능

## 설계

교체 후 `status`를 `action`으로 유지하고 `attackerAction`을 비운다.  
`turnStartTime`을 갱신해 교체 후 다시 10초를 준다.  
`manualSwitchUsedBy`와 `manualSwitchTurnStartTime`으로 같은 턴 중복 교체를 막는다.

## 의도적으로 변경하지 않은 것

- 쓰러진 펫 교체(`pending_switch`)는 기존 로직 유지
- 아라만개/벚꽃해류 등 스킬 로직 변경 없음
- 도망/아이템 처리 변경 없음

## 확인 항목

- `npm run build` 통과
- 공격권 상태에서 펫 교체 가능
- 교체 후 바로 스킬/공격 선택 가능
- 같은 턴에 두 번째 교체는 불가
- 교체 후 상대 방어 선택은 새로 가능
