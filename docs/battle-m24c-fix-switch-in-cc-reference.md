# M24c. 수동교체 행동불가 변수 보정

## 문제

M24b 적용 후 수동교체 시 `switchInCcSkipState is not defined` 오류가 발생했다.

## 원인

수동교체 처리 흐름에서 `switchInCcSkipState`를 참조하는 코드는 들어갔지만,
해당 변수를 선언하는 코드가 같은 스코프에 들어가지 않았다.

## 변경 내용

- 수동교체 로그 처리 직전에 `switchInCcSkipState`를 안전하게 선언
- 기절/경직/빙결 상태를 확인
- 행동불가 상태가 있으면 1회 소모 처리
- 기존 M24b 의도 유지:
  - 기절한 대기 펫 수동교체 가능
  - 등장 후 공격 스킵
  - 잠깐 보여준 뒤 다음 문제턴 이동

## 확인 항목

- `npm run build` 통과
- 기절한 대기 펫 수동교체 시 오류 없음
- 기본공격이 나가지 않음
- 다음 문제턴으로 넘어감
