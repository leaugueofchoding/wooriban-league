# M11. 하드 CC 공통 규칙 1차

## 목표

기절, 빙결, 속박처럼 행동을 크게 제한하는 상태를 공통 규칙으로 정리한다.

M11의 기본 원칙:

- 퀴즈 풀이 자체는 막지 않는다.
- 제한은 전투 행동 선택 단계에서 처리한다.
- 하드 CC는 방어, 회피, 도망, 교체를 제한한다.
- 같은 종류의 상태는 하드코딩 분기보다 공용 헬퍼로 처리한다.

## 1차 적용 범위

| 상태 | 방어/회피 | 도망 | 교체 | 비고 |
| --- | --- | --- | --- | --- |
| 기절 | 불가 | 불가 | 불가 | defenderAction = STUNNED |
| 빙결 | 불가 | 불가 | 불가 | defenderAction = FROZEN |
| 속박 | 불가 | 불가 | 불가 | defenderAction = BOUND |

## 적용 내용

- `getHardCcState(pet)` 공용 헬퍼 추가
- `getHardCcSwitchBlockMessage(pet)` 추가
- `getHardCcDefenseLog(pet, attackerName)` 추가
- 수동 교체에서 속박만 보던 로직을 하드 CC 공통 규칙으로 변경
- 공격자가 스킬을 고를 때 방어자가 하드 CC면 방어 선택을 기다리지 않고 자동 방어 불가 처리
- 방어자가 버튼을 눌러도 하드 CC 상태면 방어 불가 처리

## 보류: M11.2 후보

공격자 자신이 하드 CC 상태인데 퀴즈를 맞혀 공격권을 얻은 경우의 처리.

권장 방향:

- 퀴즈는 풀 수 있음
- 공격권을 얻어도 해당 턴은 행동 실패 처리
- 데미지/원소반응/스킬 SP 소모 없음
- 전용 로그: `'펫이름'은(는) 기절해서 움직이지 못했다!`
- 별도 action id를 만들 경우 애니메이션 루프와 resolution 중복 실행을 조심해야 함


## M11.5 CC 밸런스 후속 조정

- 찌릿숭이/번개 계열 경직 체감이 낮아 확률을 상향한다.
  - 따끔할퀴기: 15% → 25%
  - 뇌우: 25% → 35%
  - 정전기 방출: 15% → 22%
- 버니니 계열 재빠른 교란은 혼란이 아니라 기절로 조정한다.
  - 문제는 보이지만 제출 불가
  - 방어턴에는 무방비 판정
- 혼란 확률 분배를 조정한다.
  - 자해 45%
  - 대기펫 오폭 30%
  - 정상 공격 25%
